API Routes

This guide will outline the routes for the api, which is used for the SOSMS app.

user

GET/user/find/{user_id}

Parameters
Name Description
user_id The user id that needs to be fetched. Use 3001 for testing.
Headers
Name Description
Authorization: token Pass a token in the header. Tokens are generated on signup and login.
Response
Code Description
200 Success.
example
{
"success":true,
"payload": {
"EMAIL":"john.smith@email.com",
"FIRST_NAME":"John",
"LAST_NAME":"Smith",
"PHONE_NUMBER":"12345678",
"PREFIX":"+1",
"USER_ID":3001,
"MESSAGE":"Hey, give me a call."
"TIMER":3,
"PIN":4312,
}
}
400 Bad request.
401 Invalid path.

sms

GET/sms/send/{user_id}?location=longlat

Parameters
Name Description
user_id The user id that needs an sosms.
location (optional value) The location of user.
Headers
Name Description
Authorization: token Pass a token in the header. Tokens are generated on signup and login.
Response
Code Description
200 Success.
example
{
"success":true,
"payload": {
"results": [
{
"CONTACT_ID": 7001,
"PHONE_NUMBER":"45612890",
"PREFIX":"+1"
"FIRST_NAME":"Bruce",
"LAST_NAME":"Wayne",
"USER_ID":3001
}
]
}
}
400 Bad request.
401 Invalid path.
406 No contacts exist.

API Routes

This guide will outline the routes for the api, which is used for the SOSMS app.

validate

GET/validate/code/<phone number>

Headers
Name Description
Accept: application/json
Response
Code Description
200 Success. The app sent a validation code to the given phone number.
example
{
"success":true,
"payload": {}
}
400 Bad request.
402 Could not send message to number

POST/validate/validatecode

Headers
Name Description
Content-Type: application/json
Accept: application/json
Body
Body
{
"PHONE_NUMBER":"12345678",
"CODE":"4567"
}
Response
Code Description
200 Success.
example
{
"success":true,
"token": 1234567876543256787654,
"payload": {}
}
403 Incorrect token.

sms

POST/sms/send?location=latitude,longitude

Parameters
Name Description
location (optional value) The location of user. Latitude coordinate followed by a comma and then the longitude coordinate.
Headers
Name Description
Authorization: token Pass a token in the header. Tokens are generated on setup.
Body
Body
{
"USER":{
"FIRST_NAME":"Bruce",
"LAST_NAME":"Wayne",
"PHONE_NUMBER":"12345678",
"Message":"Hello World"
"},"
"CONTACTS":[
{
"FIRST_NAME":"Bruce",
"LAST_NAME":"Wayne",
"PHONE_NUMBER":"12345678"
}
]
}
Response
Code Description
200 Success.
example
{
"success":true,
"payload": {
"results": [
{
"FIRST_NAME":"Bruce",
"LAST_NAME":"Wayne",
"PHONE_NUMBER":"12345678"
}
]
}
}
406 No contacts in request.
